# Apartment Hunt Project Instructions

When working in this directory, follow the instructions in `apartment_hunt.md` for:
- Search parameters (schools, budget, neighborhoods, commute threshold)
- How to run a listing refresh (Step 1-5)
- HTML structure and row format
- Tag classes and pet status conventions
- Google Maps link format
- Borough mapping for JS auto-links

## Key Commands

**Refresh listings:** Follow the "How to Run a Listing Refresh" section in apartment_hunt.md. Use Playwright for StreetEasy/Nooklyn (they block plain fetches). Use search agents for RentHop, Craigslist, Zumper. Every listing needs a specific URL — no generic search pages.

**Deploy:** `git add -A && git commit -m "Refresh listings - $(date +%Y-%m-%d)" && git push origin main` — Netlify auto-deploys.

**Test links:** Start local server `python3 -m http.server 8770` from the selfprocess directory, then open `http://localhost:8770/apartment_hunt/index.html` in Playwright. Click listing links to verify they go to specific listing pages, not search pages or homepages. Click commute time links to verify they open Google Maps transit directions.

## Rules

- Every listing row must link to a specific listing page (streeteasy.com/building/..., nooklyn.com/listings/..., etc.)
- If no specific URL exists, don't include the listing
- Commute times in the ranking table link to Google Maps transit directions
- S/F pills on neighborhood rows are auto-generated by JS — don't hardcode them
- Don't change the commute summary table unless schools change
- Neighborhoods are sorted by commute quality (best worst-case first)
- Rankings table is sorted by worst-case commute time (lower = better)
